

## Revisión completa del sitio NARRAGLOBAL

### Estado general
El sitio es una single-page app con una sola ruta (`/`) y un catch-all 404. No hay rutas rotas. No hay errores en consola. La estructura es limpia.

### Problemas encontrados

#### 1. Comentarios que delatan generación automatizada
- **`src/components/clientLogos.ts` (línea 1-2):** Dice `// Auto-extracted client logos from NARRAGLOBAL HTML` y `// Each logo contains the alt text and full base64-encoded image data`. Esto es sospechoso si alguien inspecciona el código fuente. Hay que eliminarlo o reemplazarlo con algo neutral.
- **`src/integrations/supabase/client.ts` (línea 1, 8-9):** Dice `// This file is automatically generated. Do not edit it directly.` y `// Import the supabase client like this:` — comentarios de scaffolding que no deberían estar en producción.

#### 2. URL de NarraAsist expone "lovable.app"
- **`src/pages/Index.tsx` (línea 181):** El botón de NarraAsist apunta a `https://narra-assist.lovable.app/`. Si alguien inspecciona el HTML o hace hover sobre el botón, verá "lovable.app" en la barra de estado del navegador. Esto no se puede evitar sin un dominio custom para NarraAsist, pero vale la pena que lo sepas.

#### 3. `console.error` en NotFound
- **`src/pages/NotFound.tsx` (línea 9):** Tiene un `console.error("404 Error: User attempted to access...")`. Si alguien navega a una ruta inexistente, esto aparece en DevTools. Mejor quitarlo.

#### 4. Links que no llevan a ningún lado
- **Reportes (líneas 280, 291, 302):** Los tres items del índice de reportes (`<a href="#">`) apuntan a `#`, o sea no llevan a ningún lado. Si alguien clickea, solo vuelve arriba. Mejor cambiarlos a `<div>` o quitar el `href="#"`.

#### 5. CTAs de planes apuntan a `#medicion`
- Los botones "Empezar con Data →", "Empezar con Estrategia →" y "Consultar disponibilidad →" (líneas 474, 492, 512) todos apuntan a `#medicion`, que es la sección de productos, no un formulario de contacto ni WhatsApp. Deberían apuntar a un canal real (email, WhatsApp, o formulario).

#### 6. Vite config con "lovable-tagger"
- `vite.config.ts` importa `lovable-tagger`. Esto solo se carga en development mode, no aparece en producción. No es un problema real, pero si alguien inspecciona el repo de GitHub, lo verá. Es un dev dependency, así que es aceptable.

### Plan de cambios

1. **Eliminar comentarios reveladores** en `clientLogos.ts` y `supabase/client.ts`
2. **Quitar el `console.error`** de `NotFound.tsx`
3. **Cambiar los `<a href="#">`** de los reportes a elementos no-clickeables o con cursor pointer sin href
4. **Actualizar los CTAs de planes** para que apunten a un destino real (sugiero WhatsApp o mailto)
